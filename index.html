Section 1

Your Social Workspace

Before you start drafting captions, set up a project folder that makes Claude Code dramatically more useful. Think of it as giving Claude Code a cheat sheet for your brand — once it's there, every output is automatically on-voice.

The Folder Structure

Here's what a well-organized social project looks like. You don't need every file on day one — start with the brand folder and grow from there.

Terminal
~/Documents/social $ tree . ├── brand/ │ ├── voice-guidelines.md ← tone rules, do's/don'ts │ ├── hashtag-library.md ← curated tags + banned list │ └── content-pillars.md ← your 3-5 recurring themes ├── captions/ │ ├── instagram/ │ ├── tiktok/ │ └── x/ ├── video/ │ ├── scripts/ │ └── ai-prompts/ ├── community/ │ ├── comment-exports/ │ └── reply-drafts/ ├── analytics/ │ ├── ig-insights/ │ └── weekly-recaps/ └── CLAUDE.md ← project-level AI instructions

The Brand Voice File

This is the single most important file in the folder. When Claude Code reads it before drafting, every caption, reply, and script automatically matches your tone. New team members, freelancers, interns — everyone gets the same voice because it's enforced by the file, not tribal knowledge.

Claude Code
You: Create brand/voice-guidelines.md for Aceable's social presence. We're warm, motivational, and approachable. We talk to students like a supportive friend, not a corporate entity. We never use exclamation stacks (!!!). We always end captions with a soft CTA. We use second person (you/your). Include do's, don'ts, and 3 example phrases for each. Claude: I'll create that brand voice file. Created brand/voice-guidelines.md: # Aceable Social Voice Guidelines ## Core Personality Warm. Motivational. Approachable. We talk to students like a supportive friend who's been through it. ## Rules ### Do - Use second person (you, your, you're) - End every caption with a soft CTA - Keep sentences short and scannable - Examples: "You've got this." / "Your turn." / "Save this for test day." ### Don't - Stack exclamation marks (!!!) - Use corporate jargon ("leverage", "synergize") - Be condescending about test anxiety - Examples to avoid: "HUGE NEWS!!!" / "Don't miss out!!!" / "Act now or else!" ### Tone Spectrum Celebration posts → excited but grounded Study tips → encouraging, concise Testimonials → let the student's voice lead Product posts → helpful, never salesy

CLAUDE.md: Teach Claude Code Your Defaults

A CLAUDE.md file in your project root tells Claude Code your preferences every time it starts up. No more repeating "use my brand voice file" at the start of each session.

Claude Code
You: Create a CLAUDE.md that tells Claude Code: always read brand/voice-guidelines.md before drafting any content. Always check brand/hashtag-library.md before suggesting tags. Default caption format is hook → body → CTA. Target platforms: Instagram, TikTok, X, YouTube. Claude: Created CLAUDE.md: # CLAUDE.md ## Brand Rules - Always read brand/voice-guidelines.md before drafting any caption, reply, or script - Always check brand/hashtag-library.md before suggesting hashtags — filter out any tags on the banned list ## Content Defaults - Caption structure: Hook → Body → CTA - Target platforms: Instagram, TikTok, X, YouTube - When asked for "captions," generate platform-native versions for all four unless specified otherwise ## File Conventions - Save captions to captions/{platform}/ - Save video scripts to video/scripts/ - Save AI prompts to video/ai-prompts/ - Save reply drafts to community/reply-drafts/
✨
The unlock: With a CLAUDE.md in place, every conversation starts pre-loaded with your brand rules. You never have to say "read my voice file" again — it does it automatically. This is the single biggest productivity lever for repeat work.

Custom Slash Commands

Turn your most common tasks into one-word shortcuts. These live in .claude/commands/ inside your project and the whole team can share them.

/caption-ig

Reads voice guide + hashtag library, drafts an Instagram caption with hook/body/CTA structure, appends matching hashtag set.

/hashtag-set

Builds a balanced set (3 branded, 5 niche, 4 broad), cross-checks against the banned list, saves to a file.

/reply-draft

Reads a comment export CSV, drafts on-brand replies, skips spam, flags complaints for human review.

/weekly-recap

Reads IG/TikTok/YouTube analytics CSVs, generates a Slack-formatted weekly recap with top posts, growth, and recommendations.

Claude Code
You: Create a custom slash command called /caption-ig that reads brand/voice-guidelines.md and brand/hashtag-library.md, then asks me what the post is about, and generates a full Instagram caption with hook, body, CTA, and hashtag set. Claude: Created .claude/commands/caption-ig.md Your team can now type /caption-ig to: 1. Auto-load voice guidelines and hashtag library 2. Ask what the post is about 3. Generate a full caption with structure and tags Try it now!
Section 2

Content Creation Workflows

The daily grind: briefs in, captions out, hashtags researched, variations generated, all on-brand. Here's how Claude Code handles each piece — and how the pieces chain together.

1

Brief → Platform-Native Captions

The Scenario

A one-pager lands for the new real estate course launch. You need an Instagram caption (save-worthy, longer), a TikTok hook (punchy, POV), an X one-liner (under 140), and a YouTube community post. Normally this costs you 30–45 minutes of context-switching between platform voices.

Claude Code
You: Read launch-brief.md and generate four captions: Instagram (save-worthy, 40-60 words), TikTok (hook-first, POV, under 25 words), X (under 140 characters), YouTube community post (2-3 sentences, conversational). Use brand voice. Save each to captions/{platform}/. Claude: Reading launch-brief.md... Reading brand/voice-guidelines.md (via CLAUDE.md)... Instagram (58 words): "Your real estate license is closer than you think. Our new self-paced course covers every exam topic with practice questions that mirror what you'll actually see on test day. 1,200 students passed with us last year. You could be next. Save this post for when you're ready — link in bio." TikTok (18 words): "POV: you just found the course that got 1,200 people their real estate license. Link in bio." X (96 characters): "1,200 people got their real estate license last year with this course. Your turn." YouTube: "We just launched a new self-paced real estate course. It covers everything you need for exam day, and 1,200 students passed with us last year. Check the link in the description if you're ready to start." ✓ Saved to captions/instagram/, captions/tiktok/, captions/x/, captions/youtube/

One brief in, four platform-native captions out. Instagram gets the long-form save-worthy treatment. TikTok gets the hook-first POV. X gets the scannable one-liner. YouTube gets the conversational tone. No cross-posting tax — each platform gets native content in one prompt.

Balanced set, banned-tag filter, saved to a file you can paste into your scheduler. When your banned list grows (and it will), every future set benefits automatically because Claude Code reads the library fresh each time.

AI Video Creation Pipeline

AI video tools are powerful individually. The problem is coordinating them: translating one script into Runway prompts, Sora prompts, HeyGen scripts, CapCut captions — each tool speaks a different dialect. Claude Code speaks all of them.

You need a 30-second TikTok video about a student passing their real estate exam. You need: a hook (first 3 seconds), beat-by-beat voiceover, on-screen text overlays with timing, and 3 alternative hooks to test. This usually takes a copywriter an hour to structure properly.

Section 5

Social Jargon Decoder

The social-specific and AI video terms you'll encounter in this course and on the job. For general tech jargon (terminal, CLI, prompt, git), see Vibe Coding 101's Jargon Decoder.

Hook
The first 1–3 seconds of a video or the first line of a caption. Its only job is to stop the scroll. Everything else depends on it landing.
"Your hook is weak" = "Nobody's going to stop scrolling for this"
Platform-Native
Content that looks and sounds like it was made for a specific platform, not cross-posted from another. TikTok tone ≠ LinkedIn tone ≠ X tone.
"Make it platform-native" = "Don't post the IG caption on TikTok"
Brand Voice
The consistent personality of your content — tone, word choices, what you avoid. Written down in a file, it becomes enforceable by Claude Code or anyone new on the team.
"Stay on brand voice" = "Sound like us, not generic AI"
UGC
User-Generated Content. Posts, videos, or photos made by your customers or fans, not by you. Gold for authenticity, legally tricky for rights.
"Repost this UGC" = "Share something a customer made"
B-roll
Supplementary video footage layered under your main shot or voiceover. The cutaway clips that make a talking head feel dynamic.
"Add more b-roll" = "Cut away to supporting footage"
SRT
A subtitle file format. Plain text with timestamps that captioning tools (CapCut, YouTube, Descript) use to burn in captions or display closed captions.
"Export an SRT" = "Generate the caption timing file for the editor"
AI Video Prompt
A text description that tells AI video tools (Runway, Sora, Veo) what to generate. Needs technical specifics: aspect ratio, duration, camera move, mood, lighting.
"Write a Runway prompt" = "Describe the shot in a way Runway can build"
Engagement Rate
% of followers (or reach) who interacted with a post — likes, comments, shares, saves. The metric most stakeholders actually look at first.
"ER dropped" = "Fewer people are interacting with our posts"
Reach vs Impressions
Reach = unique people who saw it. Impressions = total times it was shown (same person can see it multiple times). Reach is the truer view count.
"High impressions, low reach" = "Same people keep seeing it"
Content Pillar
A recurring theme your brand posts about. Most accounts have 3–5 pillars (e.g., tips, testimonials, behind-the-scenes, product, culture). Pillars keep the feed coherent without being repetitive.
"Which pillar is this?" = "Which content bucket does this fit?"
Cadence
How often you post on each platform. IG 4x/week + TikTok daily + YouTube 1x/week is a cadence. Consistency beats volume.
"What's our cadence?" = "How often are we posting where?"
Banned Hashtags
Tags that platforms (especially Instagram) have flagged or shadow-banned. Using them can silently suppress your post's reach. Every brand should maintain a running list.
"Check the banned list" = "Make sure none of these tags are flagged"
Save Rate
% of people who bookmarked/saved your post. Instagram's algorithm weighs saves heavily. High save rate = the algorithm promotes it more.
"Optimize for saves" = "Make it reference-worthy, not just likeable"
Asset
Any piece of content — an image, a video clip, a caption, a graphic. Everything that could ship is an "asset" until it's published.
"Where's the asset?" = "Where's the file we're about to post?"
Stitch / Duet
TikTok features that let you react to or build on another creator's video. Stitch clips a segment; Duet plays side-by-side. Great for UGC and engagement bait.
"Stitch that testimonial" = "Clip the student's video and add our reaction"
CLAUDE.md
A project configuration file that Claude Code reads automatically on startup. Put your brand rules, file conventions, and default behaviors in here so you never have to repeat them.
"Update the CLAUDE.md" = "Change the default rules Claude Code follows"

Try It Yourself

Time to get your hands dirty. You're going to set up a real social media project and generate your first on-brand caption — all in about 5–7 minutes.

Exercise: Build Your Content Starter Kit

Create a project folder with a brand voice file, hashtag library, and caption template. Then prove the system works by generating a caption that uses all three.

1
Create your social project folder
Terminal
~ $ mkdir -p ~/Documents/my-social-kit && cd ~/Documents/my-social-kit
2
Launch Claude Code
Terminal
~/Documents/my-social-kit $ claude
3
Build the starter kit

Copy this prompt (replace [BRAND] with your brand name, or just use "Aceable"):

Claude Code
You: Set up a social media starter kit for [BRAND]. Create: - brand/voice-guidelines.md (tone, do's/don'ts, examples) - brand/hashtag-library.md (branded, niche, broad tags + empty banned section) - captions/template.md (hook/body/CTA structure) - CLAUDE.md (auto-read voice file before drafting, check hashtag library before tagging)
4
Review what Claude Code built

You should now have a brand/ folder with your voice rules and hashtag library, a captions/ folder with a template, and a CLAUDE.md that auto-loads your brand rules. Open them in any text editor and customize.

5
Prove it works — generate a caption

Now use the system you just built:

Claude Code
You: Draft an Instagram caption about a student who just passed the real estate exam on their first try. Use my voice guidelines, follow the caption template, and append a hashtag set from my library.

Claude Code reads your voice file and hashtag library automatically (thanks to CLAUDE.md), follows your caption template structure, and produces something on-brand. This is the pattern for every future request.

Bonus: Ask Claude Code to create a /caption-ig custom slash command that does Step 5 automatically. Then your whole team can type one word and get an on-brand Instagram caption.
