Section 1

The Terminal: It's Just a Text Box

You use text boxes all day long: caption fields, DM inputs, the search bar in your scheduler. The terminal is just another one. You type something in, and the computer responds.

You already know this
Write a caption for the new course launch...
A caption field. You type, it goes out.
This is the same idea
~ $ echo "Hello!"
A terminal. You type, it answers.

That's genuinely it. The terminal is a place where you type instructions and your computer follows them. No hacking. No coding required. Just typing.

Terminal

How to Open It

Mac

Press Cmd + Space to open Spotlight, type Terminal, and press Enter.

Or find it in Applications → Utilities → Terminal.

Windows

Press the Win key, type Terminal or PowerShell, and press Enter.

Or right-click the Start button and choose Terminal.

💚
You can't break your computer by typing in the terminal. If something looks weird or goes wrong, you can always close the window and start fresh. The terminal is not a bomb — it's a text box.

Section 3

The Jargon Decoder

Tech has a lot of jargon. Social has its own too. Here's your cheat sheet for both.

Terminal
A text box where you type instructions to your computer. That's it. It's not hacking — it's typing.
"Open the terminal" = "Open the app where you type commands"
CLI
Command Line Interface. A program you control by typing text instead of clicking buttons. Claude Code is a CLI tool.
"It's a CLI" = "You use it by typing, not clicking"
Prompt
The instruction you give to an AI. Better prompts = better results. It's like briefing an intern — the more specific, the less they guess wrong.
"Write a good prompt" = "Give clear, specific instructions"
Hook
The first 1–3 seconds of a video or the first line of a caption. Its only job is to stop the scroll. Everything else depends on it landing.
"Your hook is weak" = "Nobody's gonna stop scrolling for this"
Platform-Native
Content that looks and sounds like it was made for a specific platform, not cross-posted from another. TikTok tone ≠ LinkedIn tone ≠ X tone.
"Make it platform-native" = "Don't post the IG caption on TikTok"
Brand Voice
The consistent personality of your content — tone, word choices, what you avoid. Written down in a file, it becomes enforceable (by Claude Code or anyone new).
"Stay on brand voice" = "Sound like us, not generic AI"
UGC
User-Generated Content. Posts, videos, or photos made by your customers or fans, not by you. Gold for authenticity, legally tricky for rights.
"Repost this UGC" = "Share something a customer made"
B-roll
Supplementary video footage layered under your main shot or voiceover. The cutaway clips that make a talking head feel dynamic.
"Add more b-roll" = "Cut away to extra footage"
SRT
A subtitle file format. Plain text with timestamps that captioning software (CapCut, YouTube, Descript) uses to burn in captions.
"Export an SRT" = "Generate the caption file for the editor"
Engagement Rate
% of followers (or reach) who interacted with a post — likes, comments, shares, saves. The metric most people actually look at.
"ER dropped" = "Fewer people are interacting with our posts"
Reach vs Impressions
Reach = unique people who saw it. Impressions = total times it was seen (same person can count multiple times). Reach is the truer view.
"High impressions, low reach" = "Same people kept seeing it"
AI Video Prompt
A text description that tells AI video tools (Runway, Sora, Veo) what to generate. Needs technical specifics: aspect ratio, duration, camera move, mood.
"Write a Runway prompt" = "Describe the shot in a way Runway can build it"
Context Window
How much text an AI can "remember" in a single conversation. Think of it as Claude Code's short-term memory. Long sessions eat into it.
"The context window is full" = "The AI's memory for this chat is maxed out"
Hallucination
When an AI confidently generates information that sounds correct but is actually made up. Not lying on purpose — pattern-matching gone wrong. Always review.
"The AI hallucinated that stat" = "It made up a number that sounded real"
Token
A chunk of text that an AI processes. Roughly 1 token = 3/4 of a word. Every message you send and every response uses tokens. They're the AI's reading unit.
"That used a lot of tokens" = "That was a long conversation for the AI"
LLM
Large Language Model. The type of AI that powers Claude, ChatGPT, and similar tools. Trained on massive amounts of text so it can understand and generate human language.
"Which LLM are you using?" = "Which AI model powers your tool?"
MCP
Model Context Protocol. A standard that lets AI tools connect to external services — Slack, Drive, Notion, Meta Insights. A universal adapter for Claude Code.
"Connect via MCP" = "Let the AI talk to that other tool directly"
Directory
A folder. That's it. Developers just call folders "directories" because they like sounding fancy.
"Navigate to the directory" = "Go to the folder"
Markdown
A simple way to format text using symbols. # = heading, ** = bold, - = bullet point. Most brand voice files and content briefs are written in it.
"Write it in Markdown" = "Use these simple formatting symbols"
CSV
Comma-Separated Values. A simple spreadsheet format that everything exports to — Instagram Insights, TikTok Analytics, comment exports, content calendars.
"Export as CSV" = "Download it as a spreadsheet"
Banned Hashtags
Tags that platforms (especially Instagram) have flagged or shadow-banned. Using them can suppress your post's reach. Every brand maintains a running list.
"Check the banned list" = "Make sure none of these tags are flagged"
Content Pillar
A recurring theme your brand posts about. Most accounts have 3–5 pillars (e.g., tips, testimonials, behind-the-scenes). Pillars keep the feed coherent.
"Which pillar is this?" = "Which content bucket does this fit?"
Cadence
How often you post on each platform. IG 4x/week + TikTok daily + YouTube 1x/week is a cadence. Consistency beats volume.
"What's our cadence?" = "How often are we posting where?"
Asset
Any piece of content — an image, a video clip, a caption, a graphic. Everything that could ship is an "asset" until it's published.
"Where's the asset?" = "Where's the thing we're posting?"

Enough theory. Let's see Claude Code solve real problems your team actually faces. Each demo follows the same pattern: the scenario, the prompt, and the result.

1

Brief → Platform-Native Captions

The Scenario

You just got a one-pager brief for a course launch. Normally you spend 30 minutes translating it into an Instagram caption, a TikTok hook, and a one-liner for X — each with its own tone, length, and vibe. By the time you're done, half your morning's gone.

Save the brief into your project folder and point Claude Code at it:

Claude Code
You: Read launch-brief.md and write three captions for the course launch: one for Instagram (save-worthy, longer), one for TikTok (hook-first, punchy, POV style), and one for X (under 140 chars, shareable). Match each platform's native voice. Claude: I'll read the brief and draft all three. Reading launch-brief.md... Instagram (save-worthy): "Your real estate license is 30 days away. Our new self-paced course walks you through every exam topic, with practice questions that mirror what you'll actually see on test day. Save this post for when you're ready to start — link in bio." TikTok (hook-first): "POV: you just found the real estate course that got 1,200 people licensed last year. Yeah, this one. Link in bio. Stop scrolling." X (under 140 chars): "1,200 people got their real estate license with this course last year. Your turn." ✓ All three saved to captions/launch-captions.md

One brief in, three platform-native captions out. Each one adapts length, tone, and hook structure to the platform. No cross-posting tax. Your half-morning just became two minutes.

One source script, four tool-specific outputs, all in a project folder ready for your editor. When the hook changes, change it once in the script and re-run — every downstream artifact updates. This is the superpower: Claude Code speaks every AI video tool's prompt dialect so you don't have to.

Section 5

Tips, Tricks & Gotchas

A few things that'll make your Claude Code experience much smoother.

🎯
Be Specific
The more detail you give, the better the output. "Write a caption" is vague. "Write a 40-word Instagram caption for a real estate license pass, warm tone, end with a soft CTA to link in bio" is great. Think of it like briefing a freelancer — specificity prevents rework.
🔄
Iterate, Don't Start Over
If the first draft isn't quite right, don't restart. Just tell Claude Code what to change: "Make the hook punchier" or "Shorten to under 25 words." It's a conversation, not a one-shot. Refinement is how you get great results.
📁
Point It at Real Files
Claude Code's superpower is working with actual files. Instead of describing your brand voice, drop in brand/voice-guidelines.md and say "Read this first." Real files = enforced consistency, not vibes-based.
🤔
Ask for Explanations
If Claude Code does something you don't understand, just ask: "Explain what you just did and why you picked that hashtag set." It'll break down the reasoning. This is the fastest way to learn — and to spot when it got something wrong.
💡
Give It a Role
Tell Claude Code who it should be: "You're a TikTok growth strategist" or "Act as our brand voice editor." Setting a role tailors the tone, format, and opinions to match what you actually need.
🛠
Let It Read Before You Ask
Before asking Claude Code to write on-brand, let it read your voice file first. Say "Read brand/voice-guidelines.md, then draft the caption." Giving context up front gets dramatically better results than describing voice from memory.

Gotchas to Watch For

⚠️
It's Not Always Right
Claude Code can make mistakes — especially on specific stats, trending sounds, or recent platform changes. Always review before publishing. Trust, but verify. Nothing auto-posts unless you set it up to.
📌
File Paths Matter
Claude Code looks for files relative to where you launched it. If it can't find your brand voice file, you might be in the wrong folder. Use pwd to check, and cd ~/Documents/social to navigate to your project.
💬
Fresh Chats for Fresh Tasks
Long conversations can cause Claude Code to lose focus. Switching from drafting captions to analyzing a CSV in the same session? Type /clear and start fresh. Better results, faster responses.

Try It Yourself

Time to get your hands dirty. Here's a guided exercise you can do right now if you have Claude Code installed. Total time: about 5–7 minutes.

Exercise: Build Your Content Starter Kit

You're going to set up a project folder for your social work and let Claude Code build you a starter brand voice file, hashtag library, and caption template. Things you can actually reuse every day.

1
Open your terminal

Mac: Cmd + Space, type "Terminal", press Enter.
Windows: Press the Win key, type "Terminal", press Enter.

2
Navigate to your Documents folder
Terminal
~ $ cd ~/Documents
3
Create a project folder and go into it
Terminal
~/Documents $ mkdir my-social-kit && cd my-social-kit

mkdir creates a new folder. cd moves you into it. The && just means "do both, one after the other."

4
Launch Claude Code
Terminal
~/Documents/my-social-kit $ claude
5
Give Claude Code your prompt

Copy and paste this into Claude Code (replace [BRAND] with your actual brand or just use "Aceable"):

Claude Code
You: Set up a social media starter kit for [BRAND]. Create a folder structure with: - brand/voice-guidelines.md (tone rules, do's and don'ts, example phrases) - brand/hashtag-library.md (3 branded tags, 5 niche tags, 4 broad tags, plus an empty "banned" section) - captions/template.md (a reusable caption structure with hook, body, CTA sections) - README.md (quick guide to what's in this folder and how to use it) Use markdown formatting. Keep it practical and editable.
6
Review what Claude Code created

Claude Code creates four files across two subfolders, then shows you the structure. Every .md file is a plain text file you can open in any text editor — TextEdit, VS Code, or even Notes.

In Finder, go to Documents → my-social-kit to see the folder. Open brand/voice-guidelines.md to see what Claude Code wrote as a starting point. This is your starter kit — edit it.

7
Ask Claude Code to use your kit

Now prove it works. Try this follow-up:

Claude Code
You: Read brand/voice-guidelines.md and captions/template.md. Then draft an Instagram caption about a student passing the real estate licensing exam. Make sure it follows our voice rules and uses the template structure.

Notice how Claude Code reads your files first, then writes a caption that matches. This is the pattern for every future request.

🏆
Bonus challenge: Ask Claude Code to adapt the same caption into a TikTok hook and an X one-liner. Say: "Now give me TikTok and X versions using the same voice rules." Three platforms, one source, zero extra effort.
